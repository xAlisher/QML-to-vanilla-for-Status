# Skills

Reusable knowledge extracted from completed tasks. Each skill is a short doc capturing a technique, decision, or pattern so future sessions don't re-derive it.

## Naming convention
`{category}-{topic}.md` — e.g. `css-token-extraction.md`, `vite-project-setup.md`

## Categories
- `css-*` — CSS techniques, token systems, layout patterns
- `vite-*` — Build tooling, dev server, project scaffolding
- `qml-*` — Reading QML for HTML/CSS translation
- `design-*` — Brand/design decisions and rationale
- `workflow-*` — Process patterns (review, iteration, presentation)
